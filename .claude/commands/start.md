# 專案初始化指令

這是專案初始化流程。請依序執行以下步驟：

## 步驟 1：確認資料夾結構

首先檢查並確認專案資料夾結構已就緒：
- `assets/` - 放置參考資料、圖片、客戶提供的文件
- `docs/` - 放置專案產出文件
- `meetings/` - 放置會議記錄

## 步驟 2：引導使用者準備資料

請先顯示以下訊息給使用者：

```
🚀 開始設定新專案！

在填寫專案資訊之前，請先準備好相關文件：

📁 請將以下資料放入對應資料夾：

【assets/ 資料夾】放入：
  - 客戶 Brief 或需求文件
  - 參考資料、競品分析
  - 品牌規範、設計素材
  - 任何相關的背景資料

【docs/ 資料夾】放入：
  - 已有的專案文件（如有）
  - 提案簡報草稿（如有）

---

✅ 放好之後，請輸入「準備好了」繼續設定專案
   或直接回答下方的專案資訊問題
```

## 步驟 3：等待使用者確認後，開始收集專案資訊

當使用者確認準備好（或選擇直接開始）後，列出專案設定問題：

```
好的！請回答以下問題來設定專案：

【基本資訊】
1. 專案名稱：
2. 專案類型：(行銷活動 / 內容製作 / 產品開發 / 活動企劃 / 其他)
3. 專案目標：(一句話描述這個專案要達成什麼)

【時程】
4. 預計開始日期：
5. 預計完成日期：

【人員】
6. 專案負責人：(你自己或其他人)
7. 客戶/需求方：(這個專案是為誰做的)
8. 需要審核的人：(有誰需要 review，可填多人或無)

【範圍】
9. 主要交付物：(最後要產出什麼，例如：提案簡報、活動網站、3篇文章)
10. 預算：(選填)

【其他】
11. 其他備註：

---
請直接複製以上格式填寫，或用你自己的方式回答都可以。
```

## 步驟 4：處理使用者回覆

收到使用者的回答後：

1. **更新 `project.yaml`** - 填入專案基本資訊
2. **檢查 `assets/` 資料夾** - 如果有文件，讀取並摘要重點
3. **建立初始任務到 `tasks.md`** - 根據交付物自動拆解任務
4. **建立里程碑到 `timeline.md`** - 如果有明確時程

## 步驟 5：完成設定

設定完成後，顯示專案概覽：

```
✅ 專案設定完成！

📋 專案概覽：
- 名稱：{專案名稱}
- 目標：{專案目標}
- 時程：{開始日期} ~ {完成日期}
- 交付物：{列出交付物}

📝 已建立 {N} 個初始任務

接下來你可以：
- 輸入「目前狀態」查看專案進度
- 輸入「新增任務」增加待辦事項
- 直接告訴我任務的更新，我會幫你記錄
```
